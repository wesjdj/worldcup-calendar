# World Cup 2026 Calendar

Add every FIFA World Cup 2026 fixture to your **Outlook**, **iPhone/Apple Calendar**, or **Google Calendar** with one tap. Each event carries the venue's local kick-off time and IANA timezone, so it shows up in *your* calendar at the correct local moment, wherever you are in the world.

> 🔗 **Live site:** _enable GitHub Pages on this repo to publish — see [Deployment](#deployment)._

## Why this exists

The 2026 World Cup spans **104 matches**, **16 host cities** and **4 different time zone regimes** (US/Canada Eastern, Central, Mountain, Pacific — plus Mexico, which no longer observes DST). Manually adding fixtures is a chore and timezone math is error-prone. This site:

- Lets you **subscribe once** (`webcal://…`) so any future fixture corrections — including the round-of-32 and beyond, which currently use placeholder names — flow into your calendar automatically.
- Filters by **team**, **group**, **stage**, or **host country** before you subscribe, so you only get the matches you care about.
- Renders every kick-off in **your browser's local timezone**, with a tooltip showing the venue's local time.
- Embeds proper `VTIMEZONE` blocks in every `.ics`, so calendar clients display each match in the correct timezone (handling DST automatically).

## How users add a calendar

| Platform                  | What happens                                                              |
|---------------------------|---------------------------------------------------------------------------|
| iPhone / iPad / Mac       | Tap **Apple / iPhone** → Safari hands the `webcal://` URL to Calendar.    |
| Outlook for Windows / Mac | Tap **Apple / iPhone** — Outlook handles `webcal://` natively.            |
| Outlook.com (web)         | Tap **Outlook.com** → "Subscribe from web" dialog opens pre-filled.       |
| Google Calendar           | Tap **Google Calendar** → "Add by URL" dialog opens pre-filled.           |
| One-off import            | Tap **Download .ics** for a static file. (Won't auto-update.)             |

Subscribed feeds re-fetch every few hours. When this repo updates a fixture (e.g. round-of-16 placeholders → real teams), subscribers see the change in their next refresh — **no re-import needed**, because every event has a stable UID.

## Project layout

```
index.html              # Static site
assets/
  styles.css
  app.js                # Filter + render + per-match download
data/
  teams.json            # 48 teams, group letters, flag emoji
  venues.json           # 16 stadiums with IANA timezones
  fixtures.json         # 104 matches (generated by build-data.js)
  overrides.json        # Live patch layer (maintained by update-fixtures.js)
scripts/
  build-data.js         # Compiles ET-based schedule -> fixtures.json (applies overrides)
  build-ics.js          # Emits feeds/all.ics + per-team + per-group
  update-fixtures.js    # Pulls live results -> overrides.json
feeds/                  # Generated .ics calendar feeds
.github/workflows/pages.yml            # Build + deploy to GitHub Pages
.github/workflows/update-fixtures.yml  # Scheduled live-data refresh
```

## Local development

```bash
node scripts/build-data.js   # rebuild data/fixtures.json
node scripts/build-ics.js    # rebuild feeds/*.ics
# then open index.html via any static server, e.g.:
python -m http.server 8000
# -> http://localhost:8000
```

The site is plain HTML/JS with no build step beyond the two Node scripts (which use only the standard library).

## Deployment

The workflow at `.github/workflows/pages.yml` builds the data and feeds on every push to `main` and publishes to GitHub Pages. To enable:

1. **Settings → Pages → Build and deployment → Source: GitHub Actions**
2. Push to `main`. Site appears at `https://<owner>.github.io/<repo>/`.

## Auto-updating fixtures

This fork keeps the calendar current on its own. A scheduled workflow
(`.github/workflows/update-fixtures.yml`, every 3 hours) pulls live data from
**FIFA's public match API** and:

- **resolves knockout placeholders** (`Runner-up Group A`, `Winner Match 74`, …)
  into the real teams as results come in,
- **corrects kick-off times** if a match is rescheduled, and
- **records scores** (shown in each `.ics` event summary, e.g. `2–0`, or
  `1–1 (4–2 pens)`), and
- **enriches each event description** with goalscorers (⚽ name + minute),
  the referee, and the attendance once a match is played.

It writes only to `data/overrides.json` — a patch layer applied on top of the
hand-built schedule in `build-data.js`. The curated schedule stays the source of
truth; overrides never overwrite it, and a failed/empty response is merged
non-destructively, so already-resolved fixtures are never lost. Because every
event keeps its stable UID, subscribers see updates on their next refresh with no
re-import.

### How it works

FIFA exposes match data at:

```
https://api.fifa.com/api/v3/calendar/matches?idCompetition=17&from=2026-06-01&to=2026-07-31&count=200&language=en
```

No API key, no signup, no rate-limit budget. Each match carries a `MatchNumber`
(1–104) that maps 1:1 to this repo's fixture ids, so matching is exact rather
than fuzzy. `update-fixtures.js` reads it, maps FIFA team/stadium names to the
repo's codes, and writes `data/overrides.json`.

### Setup

Nothing to configure — just enable the workflow:

1. **Settings → Actions → General → Workflow permissions:** allow *Read and write* (so the job can commit refreshed feeds).
2. **Actions** tab → *Auto-update fixtures* → **Run workflow** to refresh immediately, or wait for the 3-hourly schedule.

Overrideable via repo *Variables* if FIFA ever shifts ids: `WC_COMPETITION_ID`
(default `17`), `WC_FROM` (`2026-06-01`), `WC_TO` (`2026-07-31`).

To run it locally:

```bash
node scripts/update-fixtures.js
node scripts/build-data.js && node scripts/build-ics.js
```

## Data sources

- Match schedule: official FIFA 2026 release (cross-referenced with NBC Sports / DAZN reporting). All ET kick-off times in the schedule are converted to UTC at build time.
- Group draw: official FIFA Final Draw, 5 December 2025.

If you spot an error in `data/fixtures.json` (or want to add knockout team names once results land), open a PR — the calendar feeds rebuild automatically.
